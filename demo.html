🛠️ Self-Healing ETL Pipeline

Best-of-N Fixers with LLM-Powered Code Repair

The Problem

📊 Upstream Schema Changes

When an upstream table column is renamed (e.g., payment_type → payment_method), the pipeline's staging SQL layer breaks silently.

⏱️ Manual Debugging

Traditional debugging requires reading error logs, inspecting schemas, and manually patching SQL — taking hours or days.

🔄 Iterative Fixes

Single-agent approaches iterate slowly, getting one shot per cycle and relying entirely on feedback to improve.

The Solution: Best-of-N Fixers

Instead of one fixer agent iterating on feedback, we parallelize the creative work and use a judge to select the strongest candidate before review.

🔍 Diagnose

Reproduces the error, inspects schema drift, and proposes acceptance criteria the Fixer must satisfy.

🔧 Fix Pool New

Spawns N=3 fixers in parallel, each with a different persona bias (Conservative, Balanced, Thorough).

🎯 Picker New

LLM judge evaluates all candidates on correctness, minimality, no dead code, and blast radius.

✅ Review

Senior engineer review checks correctness and quality. Only approves minimal, correct patches.

Architecture Highlights

📝 Typed Milestones

Each agent produces structured artifacts (Diagnosis, FixCandidate, ReviewVerdict) — not just text. Makes the trace readable like an incident thread.

🎭 Persona Diversity

Fixers get different biases: Conservative (smallest change), Balanced (clean & correct), Thorough (edge cases handled).

⚖️ Quality Gate

Optional QUALITY_MIN forces re-fix if the shipped SQL scores below threshold on 5 criteria (1-10 scale).

🔬 Deterministic Tests

15 passing tests with scripted agents — no API key needed. Full test coverage of the orchestration logic.

Quality Assessment Criteria

Post-approval, every fix is scored on five dimensions. Set QUALITY_MIN=8 to enforce a minimum bar.

✓ Correctness Contract

Preserves not_null checks and payment_type_name lookup

📏 Minimality

Smallest change that works; no unrelated rewrites

📖 Readability

Clear naming/formatting, consistent with surrounding style

🗑️ No Dead Code

No redundant fallbacks or COALESCEs masking the real fix

🛡️ Robustness

Sane null/type handling for edge cases

Code Structure

agents/ ├── contracts.py # FROZEN: shared types & milestones ├── prompts.py # System prompts (DIAGNOSER_SYS, FIXER_SYS, PICKER_SYS) ├── llm.py # LLM-backed agents (diagnoser, fixer_pool, picker, reviewer) ├── graph.py # Orchestration loop (diagnose → fix_pool → pick_best → review) ├── tools.py # Tool gateway (bruin, DuckDB, schema inspection) ├── run_demo.py # Live demo entrypoint with Weave tracing └── requirements.txt tests/ └── test_agents.py # 15 passing tests (deterministic + LLM-backed)

Benefits

  • Higher first-pass quality — Diversity of approaches increases chance of optimal solution
  • Fewer round-trips — Picker selects best candidate before Reviewer sees it
  • Parallel execution — 3× speedup in candidate generation
  • Better quality signals — Picker provides evaluation rationale for all candidates
  • Full observability — Every agent decision visible in Weave trace
  • Testable — Deterministic tests with scripted agents, no API key required
